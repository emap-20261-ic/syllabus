\begin{frame}[fragile,allowframebreaks]{Strings}

  Como já vimos, strings são arrays de caracteres. Podemos declarar
  com

  \texttt{char s[] = "HI!"}.

  Cada caractere, se está armazenado na memória, também tem um
  endereço único, e \ilcode{s} é na verdade apenas um
  \textbf{ponteiro} com o endereço do primeiro caractere.  Podemos
  acessar cada caracter como \ilcode{s[0]}, \ilcode{s[1]} etc.

  \begin{center}
    \includegraphics[width=.6\textwidth]{s_pointer.png}
  \end{center}
  
  \framebreak

  \ilcode{s} é uma variável do tipo \textbf{char*}, que é um ponteiro
  para um caractere.

  Lembre-se de que podemos ler a string inteira começando no endereço
  em \ilcode{s} e continuar lendo um caractere por vez da memória até
  chegarmos a \ilcode{\\0}.

  \framebreak

  \lstinputlisting[style=normalc,caption=src/address02.c]{src/address02.c}

  \framebreak

  Quando escrevemos \ilcode{typedef char *string;} estamos criando um
  tipo de dados personalizado para a palavra string, tornando-a
  equivalente a \ilcode{char *}.

\end{frame}

\begin{frame}[fragile,allowframebreaks]{Aritmética dos ponteiros}

  Agora que sabemos o que são strings. Podemos imprimir um caractere
  por vez com

  \lstinputlisting[style=tinyc,caption=src/address03.c]{src/address03.c}

  Quando declaramos uma string com aspas duplas, o compilador decide
  como colocar esses caracteres na memória como um array. 

  \framebreak

  Mas podemos usar os endereços diretamente

  \lstinputlisting[style=tinyc,caption=src/address04.c]{src/address04.c}

  \ilcode{*s} lê o endereço armazenado em \ilcode{s}, e \ilcode{*(s +
    1)} o endereço na memória 1 byte (char) maior.

  \ilcode{s[1]} é um \textbf{syntactic sugar}, uma abstração mais
  legível para humanos para a expressão \ilcode{*(s + 1)}.

  \framebreak

  \textbf{Pointer arithmetic} é o processo de aplicar operadores
  matemáticos à ponteiros, tratando-os como números.

\end{frame}
