## 1. Começo provocativo

```c
int a[3] = {10, 20, 30};

printf("%d\n", a[0]);
printf("%d\n", *(a + 0));
```

Pergunta: por que isso funciona?

👉 Porque em C:

```
a[i] == *(a + i)
```

## 2. O “truque” da notação de arrays

* `a[i]` **não é uma operação especial de array**
* É só açúcar sintático para aritmética de ponteiros

Inclusive, isso aqui também funciona (e assusta 😄):

```c
printf("%d\n", 1[a]);  // imprime 20
```

Porque:

```
a[i] == *(a + i) == *(i + a) == i[a]
```

## 3. Aritmética de ponteiros

```c
int a[3] = {10, 20, 30};
int *p = a;
```

* `p` aponta para `a[0]`
* `p + 1` aponta para `a[1]`

Mas o ponto importante:

👉 `p + 1` avança **sizeof(int)** bytes automaticamente


```c
printf("%p\n", (void*) p);
printf("%p\n", (void*) (p + 1));
```

## 4. Diferença fundamental: array vs ponteiro

Aqui entra o momento conceitual forte:

```c
int a[3];
int *p;
```

* `a` **é um bloco de memória fixo**
* `p` **é uma variável que guarda um endereço**

E mais importante:

```c
sizeof(a)  // tamanho do array inteiro
sizeof(p)  // tamanho do ponteiro
```

## 5. Erro clássico: “se array é ponteiro, posso fazer isso?”

Alguém vai tentar — ou você provoca:

```c
int *v = {10, 20, 30, 40};  // parece razoável... mas está errado
```

O compilador aceita com warning (ou erro), e `v[1]` produz comportamento indefinido.

**Por quê não funciona?**

`int *v` declara uma variável que **armazena um endereço** — ela não reserva espaço para nenhum elemento. A lista `{10, 20, 30, 40}` é um inicializador de **array**, não de ponteiro. O compilador trata como `int *v = 10` (o primeiro valor), que é um inteiro sendo convertido em endereço — ao acessar `v[1]`, você lê memória aleatória.

**Como fazer certo:**

```c
// Opção 1: array normal, ponteiro aponta para ele
int arr[] = {10, 20, 30, 40};
int *v = arr;

// Opção 2: compound literal (C99+) — array anônimo
int *v = (int[]){10, 20, 30, 40};
```

👉 O ponto chave: array **aloca memória** para os elementos; ponteiro apenas **guarda um endereço**. A inicialização `{...}` só funciona onde há alocação.

## 6. Passagem para funções: onde a “mágica” acontece

```c
void f(int arr[]) {
    printf("%zu\n", sizeof(arr));
}
```

E:

```c
int a[10];
f(a);
```

👉 Dentro da função:

```
int arr[]  == int *arr
```

Ou seja:

* arrays **decaiem para ponteiros**
* você perde o tamanho do array


## 7. Passagem por “referência” (entre aspas)

Aqui você conecta com a intuição dos alunos:

Em C não existe passagem por referência real — mas:

```c
void g(int *p) {
    *p = 42;
}
```

Chamando:

```c
int x = 10;
g(&x);
```

Agora com arrays:

```c
void h(int arr[]) {
    arr[0] = 99;
}
```

👉 Isso funciona porque você está passando um ponteiro para o primeiro elemento.

## 8. Amarrando tudo

Você pode fechar com essa tabela mental:

| Conceito              | O que realmente é                                            |
| --------------------- | ------------------------------------------------------------ |
| `a[i]`                | `*(a + i)`                                                   |
| array                 | bloco contínuo de memória                                    |
| nome do array         | ponteiro para o primeiro elemento (na maioria dos contextos) |
| parâmetro `int arr[]` | na verdade `int *arr`                                        |


> “C não tem arrays de verdade no sentido de abstração — tem memória contínua e ponteiros. A notação de array é só uma conveniência.”


## 9. Argumentos de linha de comando

Programas C podem receber argumentos diretamente da linha de
comando. Para isso, `main` declara dois parâmetros:

```c
int main(int argc, char* argv[])
```

- `argc` — quantidade de argumentos (inclui o próprio nome do
  programa)
- `argv` — array de strings; `argv[0]` é o nome do programa, `argv[1]`
  o primeiro argumento, e assim por diante

Pontos de atenção: `atoi("")` ou `atoi("teste")`?

## 10. Retorno de valor e código de saída

`main` retorna um `int` que sinaliza para o sistema operacional se o
programa terminou com sucesso ou com erro:

- `return 0` — sucesso
- `return 1` (ou outro valor não-zero) — falha

No shell, o operador `&&` encadeia comandos: o segundo só executa se o
primeiro retornou 0:

```bash
$ ./exit 12 && echo “Tudo bem”
hello, 12
Tudo bem
$ ./exit && echo “Tudo bem”
Missing arguments
```
