## 2. Incremento pré e pós-fixado

C tem dois operadores de incremento unitário:

```c
int a = 10;
int b = a++;  // equivale a: int b = a; a = a + 1;
int c = ++a;  // equivale a: a = a + 1; c = a;
```

- `a++` (**pós-fixado**): o valor *atual* de `a` é usado na expressão e, só
  depois, `a` é incrementado. `b` recebe 10; `a` passa a valer 11.
- `++a` (**pré-fixado**): `a` é incrementado *antes* de ser usado na
  expressão. `a` passa a valer 12; `c` recebe 12.

Em laços onde o resultado do incremento não é capturado (e.g. `n++` sozinho
no corpo de um `while`), as duas formas são equivalentes. A diferença só
importa quando o valor da expressão é usado.

## 3. Tamanho de uma string

### 3.1 Tudo na `main` — `src/length0.c`

A versão inicial percorre `argv[1]` até encontrar o byte nulo `'\0'` (que em
C também pode ser escrito como `0` ou `NUL`):

```c
int n = 0;
while (argv[1][n] != 0)
    n++;
printf("%i\n", n);
```

Strings em C terminam com `'\0'`; veja
[`semana-3/aula-20260327.md`](../semana-3/aula-20260327.md), seção "Strings e
arrays de caracteres".

### 3.2 Extraindo uma função — `src/length1.c`

Para reutilizar a lógica com qualquer string (não só `argv[1]`), extraímos
uma função:

```c
int string_length(char s[]) {
    int n = 0;
    while (s[n] != 0) n++;
    return n;
}
```

O parâmetro `char s[]` é, na prática, `char *s` — um ponteiro para o primeiro
caractere. Veja [`aula-20260401.md`](aula-20260401.md), seção 6.

### 3.3 Reutilizando a biblioteca — `src/length2.c`

A função `strlen` de `<string.h>` faz exatamente o mesmo:

```c
#include <string.h>

printf("%lu\n", strlen(argv[1]));
```

O retorno de `strlen` é do tipo `size_t` (inteiro sem sinal); o especificador
de formato correto é `%lu`. Documentação: `man string` (Linux/macOS).

## 4. Imprimindo strings caractere por caractere

### 4.1 Versão direta — `src/string0.c`

```c
for (int i = 0; i < strlen(argv[1]); i++)
    printf("%c", argv[1][i]);
```

**Problema**: `strlen` é chamada a cada iteração, percorrendo a string inteira
para recalcular o tamanho — trabalho desnecessário.

### 4.2 Versão eficiente — `src/string1.c`

```c
for (int i = 0, n = strlen(argv[1]); i < n; i++)
    printf("%c", argv[1][i]);
```

`n` é calculado **uma única vez** na inicialização do `for`. A vírgula na
cláusula de inicialização permite declarar e inicializar duas variáveis ao
mesmo tempo.

## 5. Transformando minúsculas em maiúsculas

### 5.1 Usando a tabela ASCII — `src/uppercase0.c`

Letras minúsculas (`'a'`–`'z'`) e maiúsculas (`'A'`–`'Z'`) diferem por
exatamente 32 na tabela ASCII. Subtrair 32 do código de uma minúscula produz
a maiúscula correspondente:

```c
if (argv[1][i] >= 'a' && argv[1][i] <= 'z')
    printf("%c", argv[1][i] - 32);
else
    printf("%c", argv[1][i]);
```

A abordagem funciona, mas depende de um "número mágico" (32) e de conhecer a
tabela ASCII de cor.

### 5.2 Usando `<ctype.h>` — `src/uppercase1.c`

A biblioteca padrão oferece funções para classificar e converter caracteres:

```c
#include <ctype.h>

if (islower(argv[1][i]))
    printf("%c", toupper(argv[1][i]));
else
    printf("%c", argv[1][i]);
```

- `islower(c)` — retorna verdadeiro se `c` é letra minúscula
- `toupper(c)` — converte `c` para maiúscula, se aplicável

### 5.3 Simplificando — `src/uppercase2.c`

`toupper` já retorna o próprio caractere inalterado quando ele não é uma letra
minúscula. O `if`/`else` é desnecessário:

```c
for (int i = 0, n = strlen(argv[1]); i < n; i++)
    printf("%c", toupper(argv[1][i]));
```

Sempre que possível, reutilize código da biblioteca — é mais legível, menos
propenso a erros e mais portável.
