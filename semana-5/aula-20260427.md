## 1. Motivação: arrays paralelos — `src/phonebook0.c`

Antes de introduzir `struct`, partimos de um problema concreto: uma agenda
telefônica implementada com dois arrays separados.

```c
string names[]   = {"Carter", "David"};
string numbers[] = {"+1-617-495-1000", "+1-949-468-2750"};
```

Para encontrar o telefone de "David", buscamos o índice em `names` e usamos o
*mesmo índice* em `numbers`. O alinhamento de índices é a única coisa que
mantém os dados coesos — e isso é frágil:

- Ordenar um array quebra o alinhamento com o outro.
- Adicionar uma pessoa exige inserir em *ambos* os arrays na mesma posição.
- Com três ou mais campos (nome, telefone, e-mail, endereço…) o problema se
  multiplica.

O código de busca funciona, mas o modelo mental é ruim: "pessoa" não existe
como entidade — está espalhada por vários arrays.

```c
for (int i = 0; i < 2; i++) {
  if (strcmp(names[i], "David") == 0) {
    printf("Found %s\n", numbers[i]);
    return 0;
  }
}
```

## 2. `struct`: criando tipos do domínio — `src/phonebook1.c`

`struct` agrupa campos relacionados numa única entidade. Em vez de três arrays,
temos um tipo `person` que carrega tudo junto:

```c
typedef struct {
  char name[50];
  int  idade;
  char telefone[50];
} person;
```

Agora podemos declarar variáveis do tipo `person` e inicializá-las por
posição:

```c
person a = { "Eduardo", 12, "+85 1234" };
```

Para acessar um campo, usamos o operador `.`:

```c
person b;
b.idade = 10;
strcpy(b.name, "Beatriz"); 
printf("Person: %s %d\n", b.name, b.idade);

```

O motivo de `b.name = "..."` não funcionar é o mesmo de sempre: `name` é um
array, não um ponteiro livre — sua memória já está alocada dentro de `b`. A
atribuição de string literal a um array é proibida em C; `strcpy` copia o
conteúdo caractere a caractere para o buffer existente.

## 3. As três formas de declarar uma `struct`

Em C há três variações de sintaxe, com usos distintos:

### 3.1 `struct` sem nome (anônima)

```c
struct {
  int x;
  int y;
} ponto;          // declara apenas a variável 'ponto'
```

Útil quando só precisamos de uma única instância. Não dá para declarar outras
variáveis desse tipo depois — o tipo não tem nome.

### 3.2 `struct` com tag

```c
struct Ponto {
  int x;
  int y;
};

struct Ponto p1;  // é preciso escrever 'struct Ponto' sempre
struct Ponto p2;
```

A tag `Ponto` nomeia a estrutura, permitindo declarar múltiplas variáveis. Mas
o tipo se chama `struct Ponto` — a palavra `struct` é obrigatória em toda
declaração.

### 3.3 `typedef` com `struct` (forma mais comum em C moderno)

```c
typedef struct {
  int x;
  int y;
} Ponto;

Ponto p1;         // 'struct' não é necessário
Ponto p2;
```

O `typedef` cria um alias: `Ponto` passa a ser um tipo de primeira classe,
como `int` ou `float`. É a forma mais ergonômica e a que usamos no curso.

### 3.4 `typedef` com tag (forma mista)

É possível combinar tag e `typedef` numa única declaração:

```c
typedef struct Ponto {
  int x;
  int y;
} t_ponto;
```

Agora o tipo pode ser referenciado tanto como `struct Ponto` quanto
como `t_ponto`. Isso é útil quando a struct precisa referenciar a si
mesma, onde um nó contém um ponteiro para o próximo nó do mesmo tipo:

```c
typedef struct Node {
  int value;
  struct Node *next;   // dentro da definição, 'Node' ainda não existe como alias
} t_node;              // mas 'struct Node' já é válido
```

Sem a tag, seria impossível declarar `next` — o alias `Node` só existe *depois*
que o `typedef` fecha. A tag resolve esse problema de referência antecipada.
