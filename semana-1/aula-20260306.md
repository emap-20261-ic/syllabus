### O programa hello world

Apresentar o programa `src/hello0.c`: comentários, importação do
`stdio.h` (arquivo de cabeçalho com declarações de funções e
definições de macros), a função `main` como ponto de entrada do

Funções recebem argumentos entre parênteses e retornam valores ou

O `printf` é "formatted string". Ponto-e-vírgula no final de todos os
comandos. O `return 0` indica execução sem erro. A identação organiza
o código em blocos lógicos.

Editar o código no VS Code, Replit ou GitHub Codespaces e compilar.

### Compilação com `clang`

O compilador `clang` traduz código C em um executável. Na forma mais
simples:

```bash
clang hello0.c
./a.out
```

Funciona, mas sem nenhuma verificação adicional — o compilador aceita
código que pode conter problemas. Para maior controle, passamos flags:

```bash
clang -std=c17 -Wall -Wextra -Wpedantic -Werror hello0.c -o hello
./hello
```

Em alguns ambientes, `cc` ou `gcc` ao invés de `clang`.

### Flags do `clang`

As flags nos dão controle sobre a compilação:

- `-std=c17`: define a versão do padrão C a ser usada (C17). O padrão
  C evoluiu ao longo dos anos (C89, C99, C11, C17, C23), e essa flag
  garante que o compilador use as regras da versão especificada.

- `-Wall`: ativa um conjunto amplo de avisos (warnings) — situações
  que não são erros, mas indicam código suspeito ou potencialmente
  incorreto. Exemplos: variáveis declaradas mas não usadas, uso de
  variáveis não inicializadas, comparações que são sempre verdadeiras
  ou falsas.

- `-Wextra`: ativa avisos adicionais não cobertos por `-Wall`.
  Exemplos: parâmetros de funções não utilizados, condições com
  efeitos colaterais em parênteses.

- `-Wpedantic`: emite avisos exigidos pelo padrão ISO C. Ajuda a
  garantir que o código seja portável e siga estritamente o padrão.

- `-Werror`: trata todos os avisos como erros. O compilador se recusa
  a gerar o executável se houver qualquer aviso. Isso força o
  programador a escrever código mais rigoroso.

- `-o hello`: define o nome do executável de saída. Sem esta flag, o
  padrão é `a.out`.

Recomendação: sempre compilar com todas essas flags durante o
desenvolvimento. Isso ajuda a encontrar bugs cedo e a desenvolver bons
hábitos.

### O utilitário `make`

O comando [`make`](https://www.gnu.org/software/make/) é um utilitário
que automatiza a compilação. Ele executa uma sequência de comandos
(tipicamente chamadas ao compilador) seguindo regras definidas em um
arquivo chamado `Makefile`.

```bash
make hello0
./hello0
```

Quando **não existe** um `Makefile`, o `make` tem um comportamento
*default*: para `make hello0`, ele procura `hello0.c` e chama o
compilador com regras implícitas (ex: `cc hello0.c -o hello0`).

A limitação é justamente a falta de controle: com o `make` default,
não escolhemos as flags passadas ao compilador. Para ter controle
total, criamos um `Makefile` com regras explícitas.

Exemplo de `Makefile` simples para compilar programas C:

```makefile
hello0: hello0.c
	clang -std=c17 -Wall -Wextra -Wpedantic -Werror hello0.c -o hello0
```

A primeira linha declara que o *alvo* (`hello0`) depende do arquivo
`hello0.c`. A segunda linha (indentada com TAB, não espaços!) é o
comando a executar. O `make` só recompila se `hello0.c` foi modificado
depois do executável `hello0` — isso evita recompilações
desnecessárias.

### Bibliotecas e linking

Exemplo com `src/cents1.c`: o programa usa a função `round` da
biblioteca matemática (`math.h`). No Linux, `libm` é separada e
precisa ser vinculada com `-lm`. No macOS, `libm` faz parte de
`libSystem` e é vinculada automaticamente.

```bash
clang src/cents1.c        # erro de linking no Linux
clang src/cents1.c -lm    # funciona
```

A flag `-lm` diz ao linker para vincular a biblioteca `libm`. O
padrão de nomenclatura é `-l<nome>` para vincular `lib<nome>`. Outras
bibliotecas comuns: `-lpthread` (threads), `-lncurses` (interface
terminal).

### Etapas da compilação

Compilar o código na verdade envolve quatro etapas:

1. **Pré-processamento** (pre-processing): as diretivas `#include`
   são expandidas e as declarações das funções são inseridas no
   código. Macros definidas com `#define` são substituídas. Pode-se
   ver o resultado com `clang -E hello0.c`.

2. **Compilação** (compiling): o código C pré-processado é traduzido
   para código assembly, específico para a arquitetura do processador.

3. **Montagem** (assembling): o código assembly é convertido em código
   objeto (binário), produzindo um arquivo `.o`.

4. **Vinculação** (linking): o linker junta o(s) arquivo(s) objeto com
   as bibliotecas necessárias, resolvendo referências a funções
   externas e produzindo o executável final.

O código C é transformado em código de máquina específico para a
arquitetura usada. Demonstrar com:

```bash
clang -S -mllvm --x86-asm-syntax=intel hello0.c
```

Vide `src/hello0.s`.

Mostrar representação em binário dos arquivos objeto e do executável
final após a vinculação.

### Erros

Erros ocorrem, e precisamos aprender a lidar com eles! Experimentar:

- O que ocorre se removermos o `\n`?
- O que ocorre se quebrarmos a string?
- O que ocorre se esquecermos o ponto-e-vírgula?
- O que ocorre se removermos a biblioteca?

O símbolo `\n` e o conceito de "escape".

### Um bom código C

Três dimensões de qualidade:

- **Correção**: o código resolve o problema de forma correta?
- **Design**: quão eficiente é o código ou o algoritmo?
- **Estilo**: quão bem escrito, formatado e legível é o código?

Mantenha seu código legível!
