## Primeiro programa em C (slides-15)

### O programa hello world

Apresentar o programa `src/hello0.c`: comentários, importação do
`stdio.h` (arquivo de cabeçalho com declarações de funções e
definições de macros), a função `main` como ponto de entrada do
programa.

Funções recebem argumentos entre parênteses e retornam valores ou
produzem efeitos colaterais.

O `printf` é "formatted string". Ponto-e-vírgula no final de todos os
comandos. O `return 0` indica execução sem erro. A identação organiza
o código em blocos lógicos.

Editar o código no VS Code, Replit ou GitHub Codespaces e compilar.

### Compilação

Formas de compilar:

```bash
make hello0
./hello0
clang hello0.c
./a.out
clang -std=c17 -Wall -Wextra -Wpedantic -Werror hello0.c -o hello
./hello
```

O comando `make` é um utilitário. Em alguns ambientes, `cc` ou `gcc`
ao invés de `clang`.

### Bibliotecas e linking

Exemplo com `src/cents1.c`: o programa usa a função `round` da
biblioteca matemática (`math.h`). No Linux, `libm` é separada e
precisa ser vinculada com `-lm`. No macOS, `libm` faz parte de
`libSystem` e é vinculada automaticamente.

```bash
clang src/cents1.c        # erro de linking no Linux
clang src/cents1.c -lm    # funciona
```

### Etapas da compilação

Compilar o código na verdade envolve quatro etapas:

1. Pré-processamento (pre-processing)
2. Compilação (compiling)
3. Montagem (assembling)
4. Vinculação (linking)

Mostrar o efeito do pré-processamento: as diretivas `#include` são
expandidas e as declarações das funções são inseridas no código.

O código C é transformado em código de máquina específico para a
arquitetura usada. Demonstrar com:

```bash
clang -S -mllvm --x86-asm-syntax=intel hello0.c
```

Vide `src/hello0.s`.

Mostrar representação em binário dos arquivos objeto e do executável
final após a vinculação.

### Erros

Erros ocorrem, e precisamos aprender a lidar com eles! Experimentar:

- O que ocorre se removermos o `\n`?
- O que ocorre se quebrarmos a string?
- O que ocorre se esquecermos o ponto-e-vírgula?
- O que ocorre se removermos a biblioteca?

O símbolo `\n` e o conceito de "escape".

### Um bom código C

Três dimensões de qualidade:

- **Correção**: o código resolve o problema de forma correta?
- **Design**: quão eficiente é o código ou o algoritmo?
- **Estilo**: quão bem escrito, formatado e legível é o código?

Mantenha seu código legível!
