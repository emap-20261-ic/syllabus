## Decisão: `if-then-else` (slides-23)

### Sintaxe

```c
if (condição)
    instrução-then;
else
    instrução-else;
```

Tanto a parte `then` quanto a parte `else` são *statements*: podem ser
uma instrução simples ou um bloco entre chaves. A parte `else` é
opcional.

### Exemplo: area.c

O programa calcula a área de um círculo a partir do raio informado
pelo usuário.

Pontos a destacar:

- `const`: o valor não pode ser alterado após a inicialização — uma
  tentativa de atribuição gera erro de compilação. Por convenção,
  nomes de constantes são escritos em maiúsculas.
- Validar a entrada **antes de usar**: `if (raio < 0)` — o programa
  encerra com erro antes de tentar calcular.
- `return 1` (ou qualquer valor não-zero) em `main`: convenção de
  erro. `return 0`: convenção de sucesso.
- Declaração de variável no meio do bloco (`double area = ...`),
  válido mas não necessariamente uma boa prática.
- `%lf` para ler `double` com `scanf`; `%lf` (ou `%f`) para imprimir
  com `printf`.

### Exemplo: agree.c

O programa lê um caractere e verifica se o usuário concordou.

Pontos a destacar:

- O tipo `char` representa um caractere (1 byte); lê-se com `scanf("%c", &c)`.
- Literais de `char` usam aspas simples: `'Y'` (não `"Y"`, que seria uma string).
- O operador `==` testa igualdade (diferente do `=` de atribuição).
- Os operadores lógicos `||` (ou) e `&&` (e) combinam condições.
- O `else` aninhado permite tratar mais de dois casos.
- Retornar `1` de `main` sinaliza erro ao sistema operacional.
- Vale testar com: `Y`, `y`, `N`, `n`, e outro caractere qualquer.
- O que acontece se o usuário digitar `yes`? Apenas o `'y'` é lido pelo `scanf`.

### Exemplo: parity0.c

O operador `%` (módulo) retorna o resto da divisão inteira. Se o resto
por 2 é zero, o número é par.

Pontos a destacar:

- `7 % 2 = 1`, `4 % 2 = 0`: exemplos básicos do operador módulo.
- `n % 2 == 0` → `n` é par; `n % 2 != 0` → `n` é ímpar.
- Cuidado com negativos: `-3 % 2` em C89 dependia da implementação; em
  C99+ é definido como truncamento em direção a zero (resultado `-1`).
- `if/else` sem chaves: válido para uma única instrução, mas usar
  chaves por hábito evita erros sutis (ver seção sobre chaves
  opcionais adiante).

### Exemplo: raiz.c — `if-else` aninhado

O programa `src/raiz.c` resolve equações de segundo grau. Ilustra o `if-else`
aninhado para cobrir três casos: delta negativo, delta zero e delta positivo.

Pontos a destacar:

- `#include <math.h>` para `pow()` e `sqrt()`; no Linux, compilar com
  a flag `-lm`.
- Leitura de `double`: usar `scanf` com `%lf` (não `%f`); `printf`
  aceita `%lf` ou `%f`.
- Estrutura `if / else if / else`: exatamente um dos ramos é executado
  e o outro ignorado.
- Variáveis declaradas dentro do bloco (`double x`, `double x1`,
  `double x2`) têm escopo local ao bloco: deixam de existir quando o
  bloco termina.
- Comparar `delta == 0` com `double` é problemático por causa da
  imprecisão do ponto flutuante — vale mencionar brevemente; soluções
  robustas usam uma tolerância, como `fabs(delta) < 1e-9`.
- `return 0` ao final de `main`: convenção de sucesso.

### Tratamento de erros de entrada e a instrução `return`

Quando o `scanf` não consegue consumir a entrada esperada (por
exemplo, o usuário digita uma letra onde se espera um número), ele
retorna um valor diferente do número de itens lidos com sucesso e o
buffer de entrada fica com dados não consumidos. Uma estratégia
simples é verificar o valor de retorno do `scanf` e encerrar o
programa com `return 1`:

```c
int n;
if (scanf("%d", &n) != 1) {
    printf("Entrada inválida.\n");
    return 1;
}
```

O `return 1` em `main` sinaliza erro ao sistema operacional
(convenção: `0` indica sucesso, qualquer outro valor indica
falha). Vale destacar o que `return` faz: ele devolve o valor indicado
**e encerra a execução de `main` imediatamente**. Tudo que vier depois
no código não será executado.

Isso é útil como estratégia de saída antecipada: ao detectar uma
condição de erro logo no início, encerramos o programa antes de
executar o restante, sem precisar de `if-else` aninhado. O programa
abaixo, por exemplo, imprime "ok" apenas se a leitura do número foi
bem-sucedida — caso contrário, encerra na linha do `return 1` e a
última `printf` nunca é alcançada:

```c
int n;
if (scanf("%d", &n) != 1) {
    printf("Entrada inválida.\n");
    return 1;    // encerra aqui; nada abaixo executa
}
printf("ok\n");  // só chega aqui se scanf teve sucesso
```

Outra estratégia, usada em `parity0.c`, é inicializar a variável com
um valor sentinela inválido e verificar após a leitura:

```c
int n = -1;
scanf("%d", &n);
if (n < 0)
    return 1;
```

### A estrutura recursiva da linguagem

Uma observação importante sobre a gramática de C: a linguagem é definida de
forma recursiva. Um *statement* (`if`, `for`, `while`) tem como parte de sua
sintaxe outros *statements*, que por sua vez podem ser compostos por novos
*statements*, e assim por diante.

- Um *bloco* é uma sequência de zero ou mais *statements* entre `{` e `}`.
- Um `if-then-else` é ele próprio um *statement* — portanto pode aparecer
  dentro do ramo `then` ou `else` de outro `if` (como em `raiz.c`).
- Essa recursividade permite representar qualquer estrutura de controle,
  por mais complexa que seja.

Isso tem duas consequências práticas que vale destacar:

1. **Estrutura estática vs. execução dinâmica**: há duas formas distintas de
   "contar instruções" em um programa, e é importante não confundi-las.

   - **Estática**: conta-se quantas instruções existem no *código
     fonte*, sem executar o programa. É uma propriedade do texto do
     programa, independente de qualquer entrada. Em `raiz.c`, por
     exemplo, existem instruções nos três ramos do `if-else` aninhado
     — todas elas fazem parte do código.

   - **Dinâmica**: conta-se quais instruções são *de fato executadas*
     para uma entrada específica. Isso só se sabe em tempo de
     execução, quando as condições são avaliadas. Se delta for
     negativo, apenas o ramo correspondente é executado; os outros
     dois são ignorados naquela execução.

   A distinção importa: perguntar "quantas instruções o programa tem?" é uma
   pergunta estática. Perguntar "quantas instruções foram executadas quando
   entrei com `a=1, b=0, c=-1`?" é uma pergunta dinâmica — e a resposta muda
   conforme a entrada.

2. **Rastrear o fluxo de execução**: um exercício útil é, dado um programa e uma
   entrada específica, seguir manualmente quais instruções são executadas, na
   ordem em que o computador as executaria. Isso é uma análise dinâmica: o
   percurso pelo código depende da entrada.

## Repetição (slides-24)

### `do-while`

```c
do {
    instrução;
} while (condição);
```

O bloco é executado **pelo menos uma vez**, e depois a condição é testada. Se a
condição for verdadeira, repete; se for falsa, sai do laço.

Uso típico: validar entrada do usuário — queremos sempre pedir pelo menos uma
vez:

```c
int n;
do {
    printf("width: ");
    scanf("%d", &n);
} while (n < 1);
```

Esse padrão aparece em `src/mario2.c`: o programa só aceita largura positiva.

### Exemplo: guess.c — `do-while` e `#define`

O programa pede um número ao usuário até que ele acerte o valor alvo.

Pontos a destacar:

- `#define target 10`: macro de pré-processador (não é variável!). O
  pré-processador substitui `target` por `10` antes da compilação.
  Convenção: nomes de macro em maiúsculas.
- `do-while` faz sentido aqui: queremos sempre pedir pelo menos uma vez
  antes de verificar — diferente do `while`, que testa antes de entrar.
- A variável `n` conta as tentativas: incrementada a cada iteração.
- Testar: o que acontece se o usuário acertar na primeira tentativa?

Pontos a destacar em `mario2.c`:

- O `do-while` pede o tamanho ao menos uma vez e repete enquanto `n < 1`.
- Vale testar com entradas inválidas: `0`, `-1`, `100`, `'a'`, `"a10"`, `"10a"`.
- Comportamento do `scanf` com entrada não-numérica: tenta ler um
  inteiro; se falhar (ex.: `'a'`), retorna `0` e o valor em `n` pode
  ser lixo; o caractere não consumido fica no buffer e pode causar
  loop infinito. Uma solução robusta será vista mais adiante.
- Após o `do-while`, `n` é garantidamente ≥ 1 — o `for` seguinte pode
  usá-lo com segurança.
- `printf("?")` sem `\n`: imprime na mesma linha. `printf("\n")` ao
  final quebra a linha.

### `for`

```c
for (inicialização; condição; atualização)
    instrução;
```

As três partes são separadas por `;`:

1. **inicialização** — executada uma única vez, antes do laço começar.
2. **condição** — testada antes de cada iteração; se falsa, o laço termina.
3. **atualização** — executada ao fim de cada iteração.

Exemplo: imprimir os números de 0 a n-1:

```c
for (int i = 0; i < n; i++)
    printf(" %d ", i);
```

O `for` é equivalente ao `while` com inicialização explícita:

```c
int i = 0;
while (i < n) {
    printf(" %d ", i);
    i++;
}
```

A preferência pelo `for` é de estilo: ele agrupa em uma linha a variável de
controle, a condição de parada e o incremento, tornando mais fácil entender
a estrutura de repetição.

#### Escopo da variável de controle

Quando a variável é declarada no próprio `for` (`for (int i = 0; ...)`), ela
existe apenas dentro do laço. Ao declarar fora (`int i; for (i = 0; ...)`), ela
continua acessível após o término do laço — o que pode ser útil para inspecionar
o valor final de `i`, como mostra `mario2.c`:

```c
printf("\no ultimo i foi %d\n", i);
```

### Equivalência entre as três estruturas de repetição

As três estruturas — `for`, `while` e `do-while` — têm o mesmo poder expressivo:
qualquer laço escrito com uma delas pode ser reescrito com as outras duas sem
alterar o comportamento.

A equivalência mais direta é entre `for` e `while`. Um `for (init; cond; update)`
equivale exatamente a:

```c
init;
while (cond) {
    corpo;
    update;
}
```

Reciprocamente, um `while` sem inicialização ou atualização explícitas se escreve
como `for` com esses campos em branco: `for (; cond; ) { corpo; }`.

O `do-while` exige um passo a mais para ser simulado, porque executa o corpo
antes do primeiro teste. A equivalência passa pela lógica "execute sempre, saia
quando a condição falhar":

```c
for (;;) {
    corpo;
    if (!condição) break;
}
```

O `for` não é, portanto, mais *poderoso* que os outros — é mais *compacto e
expressivo*: reúne inicialização, condição e atualização em uma linha, deixando
visível de imediato a estrutura do laço. A escolha entre as três formas é de
estilo e legibilidade: `for` quando o número de iterações é conhecido de antemão;
`while` quando há apenas uma condição de parada sem atualização óbvia; `do-while`
quando o corpo deve executar pelo menos uma vez.

### O que o compilador vê

Para o compilador, as três estruturas colapsam no mesmo padrão primitivo. Antes
de gerar código de máquina, o GCC e o Clang convertem os laços para
representações intermediárias (GIMPLE e LLVM IR, respectivamente) que não
possuem `for`, `while` ou `do-while` — apenas testes e saltos. O padrão gerado é
conceitualmente equivalente a:

```c
init;

loop:
  if (!cond) goto fim;
  corpo;
  update;
  goto loop;

fim:
```

Para o `do-while`, o teste aparece apenas depois do corpo, eliminando o `if`
inicial:

```c
loop:
  corpo;
  if (cond) goto loop;
```

No nível do assembly, "testar" vira uma instrução de comparação (`cmp`) e
"saltar" vira um salto condicional (`jge`, `jne`, …). Em LLVM IR, o equivalente
são blocos básicos conectados por instruções `br` (branch). A equivalência entre
as três estruturas de alto nível reflete diretamente esse fato: todas mapeiam
para o mesmo mecanismo primitivo. As palavras-chave `for`, `while` e `do-while`
são conveniência para o programador; para o compilador, tudo vira grafo de fluxo
de controle com condições e saltos.

### A regra das chaves opcionais

Quando o corpo de um `if`, `else`, `while`, `do` ou `for` é composto por
**uma única instrução**, as chaves são opcionais:

```c
if (x > 0)
    printf("positivo\n");   // sem chaves: ok

if (x > 0) {
    printf("positivo\n");   // com chaves: também ok
}
```

Com **duas ou mais** instruções, as chaves são **obrigatórias**:

```c
if (x > 0) {
    printf("positivo\n");
    count++;
}
```

Omitir as chaves quando há uma instrução apenas é válido, mas pode induzir
a erros sutis. Um erro clássico:

```c
if (x > 0)
    printf("positivo\n");
    count++;               // sempre executa, mesmo com x <= 0!
```

A indentação engana o leitor, mas o compilador ignora indentação. A
instrução `count++` sempre executa porque não está dentro de um bloco `if`.
Por isso muitos guias de estilo recomendam usar chaves sempre.

## Exemplos adicionais

### meow0.c → meow1.c → meow2.c → meow3.c → meow4.c

A sequência de arquivos `src/meow*.c` ilustra a evolução de um programa
simples que imprime "meow" três vezes, passando de repetição explícita para
laço `while`, depois `for`, e por fim extração para uma função:

- **meow0.c**: três `printf` explícitos. Motivação: e se quisermos 100 meows?
  Código duplicado é difícil de manter — mudar `"meow"` para `"woof"` exigiria
  alterar em múltiplos lugares.
- **meow1.c**: usa `while`. O índice `i` começa em 0 (convenção em C). A
  condição `i++ < 3` usa incremento pós-fixo: avalia `i` (0, 1, 2) e só depois
  incrementa — o laço executa exatamente 3 vezes.
- **meow2.c**: usa `for`, mais idiomático quando o número de iterações é
  conhecido. A variável `int i` declarada no `for` tem escopo limitado ao laço
  (C99+).
- **meow3.c**: extrai a impressão para uma função `meow(void)`.
- **meow4.c**: a função recebe o número de repetições como parâmetro:
  `meow(int n)`.

### scores0.c — divisão inteira e especificador errado

Antes de compilar, vale perguntar: *o que será impresso?*

`(score1 + score2 + score3)` é `int + int + int = int`. Dividir por
`3.0` (um `double`) promove o resultado para `double` — até aqui a
aritmética está correta. O problema é que o código usa `%d` para
imprimir, e `%d` espera um `int`. Passar um `double` onde se espera
`int` é **comportamento indefinido**: a saída será lixo, não o valor
calculado.

Compilar com `-Wall` faz o compilador emitir um aviso sobre o
especificador de formato incorreto.

Há duas maneiras de corrigir:

```c
// opção 1: especificador correto para double
printf("Average: %f\n", (score1 + score2 + score3) / 3.0);

// opção 2: divisão inteira, aceitar o truncamento
printf("Average: %d\n", (score1 + score2 + score3) / 3);
```

**Lição**: o especificador de formato **deve** corresponder ao tipo da
expressão passada ao `printf`. O compilador não faz essa verificação
automaticamente — mas `-Wall` ajuda a detectar o erro.

O tema da divisão inteira volta em `calculator3.c` (slides-24), desta
vez com foco no *cast* explícito como solução.

### Exemplo: cents.c — imprecisão de ponto flutuante

O programa converte um valor em dólares para centavos multiplicando por
100 e truncando para `int`.

Antes de executar, perguntar aos alunos: *o que será impresso para 4.20?*

Pontos a destacar:

- `0.99` → 99 ✓, `1.23` → 123 ✓, `4.20` → 419 ✗ (esperado: 420)
- Causa: `4.20` não tem representação exata em binário — é armazenado
  internamente como `4.19999980...`
  `4.19999980 × 100 = 419.99998...` → truncado para `int` → 419
- Não é um bug do C; é uma limitação fundamental do ponto flutuante (IEEE 754)

O programa `cents1.c` corrige o problema de `cents0.c` usando
`round()`.

Pontos a destacar:

- `#include <math.h>` e flag `-lm` para compilar
- `round(x)`: arredonda para o inteiro mais próximo (retorna `double`)
- `round(4.19999... × 100) = round(419.9999...) = 420.0` → `int`: 420
- Testar com os mesmos valores de `cents0.c`: agora `4.20` → 420 ✓
- Lição: ao trabalhar com valores monetários, cuidado com `float`;
  alternativa robusta: sempre trabalhar em centavos (inteiros)

### Exemplo: mario.c

A família de arquivos `src/mario*.c` mostra uma progressão: de
`mario1.c` (tamanho fixo) para `mario2.c` (entrada do usuário com
validação) e `mario3.c` (grade n×n com laços aninhados).

**mario1.c**: imprime 4 blocos fixos em uma linha. O número `4` está
*hard-coded* — para mudar o tamanho seria preciso alterar o
código. `printf("#")` sem `\n` mantém tudo na mesma linha; o `\n` vem
só ao final.

**mario3.c**: usa dois `for` aninhados para imprimir uma grade n×n.

Pontos a destacar:

- Loop externo (`i`): controla as linhas; ao final de cada iteração,
  `printf("\n")` quebra a linha.
- Loop interno (`j`): controla as colunas; `printf("#")` sem `\n`
  imprime na mesma linha.
- Cada variável de loop tem seu próprio escopo (`i` e `j` são
  independentes).
- Testar: `n=3` → grade 3×3; `n=5` → grade 5×5.
- Desafio: como imprimir apenas o triângulo inferior? Dica: mudar a
  condição do loop interno para `j < i + 1`.
