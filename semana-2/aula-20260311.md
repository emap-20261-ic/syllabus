## A função `printf` (slides-22)

```c
printf(<format-string>, e1, e2, ...);
```

A `format-string` é uma string que pode conter:
- Texto literal
- **Especificadores de formato**: substituídos pelos valores das expressões

| Especificador | Tipo            |
| ------------- | --------------- |
| `%d`          | `int`           |
| `%ld`         | `long`          |
| `%f`          | `float`         |
| `%lf`         | `double`        |
| `%c`          | `char`          |
| `%s`          | string          |

**Formatação**: `%8.2f` — campo de 8 caracteres, 2 casas decimais; `%-3s` —
alinhado à esquerda em campo de 3. Exemplo:

```c
printf("%-3s %8s\n"  , "Var", "Val");
printf("%-3s %8.2f\n", "x"  , 10.222);
printf("%-3s %8.2f\n", "y"  , 20.33);
printf("%-3s %8.2f\n", "z"  , 30E2);
```

Saída:
```
Var      Val
x      10.22
y      20.33
z    3000.00
```

**Caracteres de escape** (sequências iniciadas por `\`):

| Sequência | Significado         |
| --------- | ------------------- |
| `\n`      | nova linha          |
| `\t`      | tabulação           |
| `\"`      | aspas duplas        |
| `\'`      | aspas simples       |
| `\\`      | barra invertida     |
| `\a`      | beep                |

## A função `scanf` (slides-22)

```c
scanf(<format-string>, addr1, addr2, ...);
```

Diferença crucial em relação ao `printf`:

- `printf` imprime o **valor** das variáveis
- `scanf` **altera o conteúdo** de variáveis, ou seja, **escreve na memória**
- Por isso sempre passamos um **endereço de memória**: o operador `&`

```c
int x;
scanf("%d", &x);   // correto: passa endereço de x

scanf(x);          // erro
scanf("%d", x);    // erro: passa o valor, não o endereço
```

Exemplo completo com múltiplas variáveis e formatação no scanf:
```c
int n;
float m;
char c;
scanf("%d ",  &n);
scanf("%c ",  &c);
scanf("/%f/", &m);
printf("n:[%d] c:[%c] m:[%.3f] \n", n, c, m);
```
Entrada: `123   a    /4.5/` → saída: `n:[123] c:[a] m:[4.500]`

### Exemplo: calculator1-aula.c

Abrir `src/calculator1-aula.c`. Compilar e testar. Pontos a destacar:
- `scanf` com `&`: o `&` passa o endereço de memória de `x`
- Padrão prompt + scanf em sequência
- `%i` é equivalente a `%d` para `int`

### Divisão inteira: scores0-aula.c

Abrir `src/scores0-aula.c`. Antes de compilar, perguntar aos alunos: **o que
será impresso?**

O compilador emite um warning: `format specifies type 'int' but the argument has
type 'double'`. O `%d` espera um `int`, mas `(score1+score2+score3)/3.0` é um
`double`. A saída será lixo.

Lições:
1. A divisão `int / int` **trunca** o resultado (descarta a parte fracionária)
2. `int / 3.0` é `int / double` → promovido para `double`
3. O especificador de formato deve corresponder ao tipo da expressão
4. Usar `-Wall` para ver os warnings

## Condicionais e expressões lógicas (slides-23)

A estrutura `if/else` em C:

```c
if (<condição>) {
    // bloco executado se condição é verdadeira
} else if (<outra condição>) {
    // ...
} else {
    // bloco executado se nenhuma condição foi verdadeira
}
```

Se o bloco tiver apenas uma instrução, as chaves `{}` são opcionais — mas boa
prática é sempre usar.

### Equação do segundo grau: raiz-aula.c

Exemplo completo e realista: ler coeficientes, calcular delta, decidir com
`if/else if/else`. Pontos a destacar:
- `#include <math.h>` e flag `-lm` para linkar a biblioteca
- `pow(b, 2)` vs `b * b`
- Declaração de variável dentro do bloco (`double x = ...`) — escopo local
- `delta == 0` com ponto flutuante: discutir brevemente (voltaremos em slides-24)

### Paridade: parity0-aula.c

Simples e direto. O operador `%` (módulo/resto da divisão inteira):
- `n % 2 == 0` → n é par
- `n % 2 != 0` → n é ímpar (ou usar `n % 2 == 1`, mas cuidado com negativos)

### Operadores relacionais e lógicos

| Tipo         | Operadores                                |
| ------------ | ----------------------------------------- |
| Relacionais  | `>`, `<`, `>=`, `<=`, `==`, `!=`          |
| Lógicos      | `!p` (not), `p \|\| q` (or), `p && q` (and) |

Em C, `0` representa **falso** e qualquer valor não-nulo representa **verdadeiro**.
`stdbool.h` define `true` e `false` como constantes.

### Agree: agree-aula.c

Pontos a destacar:
- Tipo `char`, leitura com `scanf("%c", &c)`
- Literais de caractere: **aspas simples** `'Y'` vs **aspas duplas** `"Y"` (string)
- Operador `||`: basta uma condição ser verdadeira

### Área do círculo: area-aula.c

Pontos a destacar:
- `const double PI = 3.141592` — valor que não muda após inicialização
- Validação de entrada antes de calcular: `if (raio < 0)`
- `return 1` para indicar erro; `return 0` para sucesso
- Declaração de variável no meio do bloco (C99+): `double area = ...`

## Repetição (slides-23)

### Motivação: meow0-aula.c

Três `printf` idênticos — problema: e se quisermos 100 meows? Péssimo estilo.

### while: meow1-aula.c

```c
while (<condição>) {
    <corpo>
}
```

O `i++` é incremento pós-fixo: avalia `i`, depois incrementa. Equivalente a
`i = i + 1`.

### for: meow2-aula.c

```c
for (<init>; <condição>; <passo>) {
    <corpo>
}
```

O `for` é normalmente preferível ao `while` quando o número de iterações é
conhecido. Três partes: inicialização, condição de continuação, passo.

### Loop infinito

```c
while (true) {
    printf("meow\n");
}
```

Requer `#include <stdbool.h>`. Para terminar: `CTRL+C`.

### do-while: guess-aula.c

```c
do {
    <corpo>
} while (<condição>);
```

Diferença do `while`: o corpo é executado **pelo menos uma vez**, depois a
condição é verificada.

`#define target 10` — macro de pré-processador: substitui o texto `target` por
`10` antes de compilar. Não é uma variável.

## Tipos, overflow e precisão (slides-24)

### Integer overflow: calculator1 revisitado

Testar com `x = 2000000000` e `y = 2000000000`. O resultado esperado seria
4.000.000.000, mas um `int` de 32 bits comporta no máximo 2.147.483.648 (2³¹).
O resultado será negativo: **integer overflow**.

### calculator2.c

Mudar o tipo para `long` (64 bits). O overflow deixa de ocorrer.
`%li` é o especificador para `long`.

### Divisão inteira: calculator3.c

Em `scores0.c` já vimos divisão inteira pelo ângulo do *especificador errado no
printf*. Aqui revisitamos o mesmo tema, agora pelo ângulo do *cast explícito*
como solução.

```c
int x, y;
float z = x / y;   // int/int = int, depois converte para float
```

A divisão de dois `int` produz um `int` (truncado), mesmo que o resultado seja
atribuído a um `float`. Para obter a divisão real:

```c
float z = (float) x / (float) y;   // cast explícito
// ou
float z = (float) x / y;           // basta um dos operandos ser float
```

Isto é o que chamamos de **imprecisão de ponto flutuante**: não é possível
representar todos os números reais com um número finito de bits.

```
2 / 3 = 0.66666668653488...   (não 0.666...)
1 / 10 = 0.10000000149011...  (não 0.1)
```

### cents0-aula.c

Problema: converter dólares para centavos.
- `4.20` é representado internamente como `4.199999...`
- `4.199999... * 100 = 419.999...` → convertido para `int`: `419` (truncamento)

### cents1-aula.c

Solução: `round(amount * 100)` da `<math.h>`.
- `round()` arredonda para o inteiro mais próximo antes de converter
