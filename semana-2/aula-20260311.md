## A string de formato do `printf` e `scanf`

Tanto `printf` quanto `scanf` recebem como primeiro argumento uma **string de
formato** — um texto entre aspas que mistura caracteres literais com
**especificadores de conversão**, que começam com `%`:

```c
printf("x = %d\n", x);   // "%d" é um especificador: lê um int e o formata
```

O `%` dentro de uma string de formato **não é o operador de módulo**. É um
marcador sintático da mini-linguagem de formatação do `printf`/`scanf`: indica
que ali deve vir um valor de um certo tipo. Os especificadores mais comuns são:

| Especificador | Tipo        | Exemplo               |
|---------------|-------------|-----------------------|
| `%d` ou `%i`  | `int`       | `printf("%d", 42)`    |
| `%f`          | `float`     | `printf("%f", 3.14f)` |
| `%lf`         | `double`    | `scanf("%lf", &x)`    |
| `%c`          | `char`      | `printf("%c", 'A')`   |
| `%%`          | literal `%` | `printf("100%%")`     |

O `%` como **operador** e o `%` dentro de uma **string** são coisas distintas:

```c
int r = 10 % 3;          // operador módulo: r recebe 1
printf("r = %d\n", r);   // %d dentro da string: especificador de formato
```

No primeiro caso, `%` está no código C e é interpretado pelo compilador como
operador aritmético. No segundo, está dentro das aspas — é apenas um caractere
da string, interpretado por `printf` em tempo de execução segundo suas próprias
regras. O compilador C não vê o `%` da string como operador; ele vê a string
inteira como um valor literal.

## A função `scanf` (slides-22)

A diferença crucial em relação ao `printf`: `scanf` **escreve na
memória**, então sempre passamos um **endereço** com o operador `&`.

```c
int x;
scanf("%d", &x);   // correto
scanf("%d", x);    // erro: passa o valor, não o endereço
```

### Como o `scanf` lê a entrada

O `scanf` opera em dois passos:

1. **Ignora espaços em branco iniciais** — espaços, tabs e quebras de
   linha são pulados antes de começar a ler.
2. **Lê até encontrar algo que não pertence ao tipo** — e deixa o
   restante no buffer para a próxima chamada.

Isso significa que múltiplos valores podem ser digitados separados por
qualquer quantidade de espaços ou quebras de linhas, e o `scanf` vai
encontrá-los corretamente.

### Exemplo: calculator1.c

Abrir `src/calculator1.c`. Compilar e testar. Pontos a destacar:

- O `&` no `scanf` passa o endereço de memória de `x`
- Padrão prompt + scanf em sequência
- `%i` é equivalente a `%d` para `int`
- `int / int = int` (truncado): a divisão acontece **antes** da conversão para `float`.

- Solução: *cast* explícito antes da divisão:

## Representação de ponto flutuante

Ao mudar as variáveis de `int` para `float` no `calculator1-aula.c`,
observamos:

```
% ./a.out
x: 12.3
y: 12
12.300000 + 12.000000 = 24.299999
```

Por que `24.299999` e não `24.300000`?

Computadores representam números em **binário**. Assim como `1/3` não
tem representação decimal exata (0.333...), `12.3` não tem
representação binária exata — é uma dízima binária. O valor armazenado
é uma aproximação muito próxima, mas não exata. Quando somamos duas
aproximações, o erro se acumula e aparece nas casas decimais menos
significativas.

`float` tem precisão de ~7 dígitos decimais; `double`, de ~15. Por
isso `double` é preferido na maioria dos casos práticos — mas o
problema não desaparece, apenas diminui.

Para confirmar, testar com `double`:

```c
double x, y;
scanf("%lf", &x);
scanf("%lf", &y);
printf("%.10f + %.10f = %.10f\n", x, y, x + y);
```

A imprecisão ainda existe, só aparece mais tarde nas casas decimais.

### Como os floats são representados: IEEE 754

O esquema de representação de `float` (32 bits) e `double` (64 bits) é
padronizado pelo padrão **IEEE 754**. Um `float` divide seus 32 bits
em três campos:

- **1 bit de sinal**: 0 para positivo, 1 para negativo.
- **8 bits de expoente**: representa a potência de 2 que escala o
  valor (armazenado com um viés de 127 para evitar negativos).
- **23 bits de mantissa**: a parte fracionária significativa do
  número em binário.

O valor é calculado como: $(-1)^{\text{sinal}} \times
1.\text{mantissa}_2 \times 2^{\text{expoente}-127}$.

Isso é análogo à notação científica decimal ($1.23 \times 10^4$), mas
em base 2. A consequência direta é que apenas frações que são somas
de potências de $2^{-k}$ são representadas exatamente — a maioria
dos decimais comuns não é.

Assim como inteiros têm um intervalo máximo (overflow), floats têm uma
**precisão** limitada pela mantissa. Cada tipo de dado em C tem seu
próprio esquema de representação em bytes: o compilador sabe como
interpretar os bytes de uma variável porque o programador declarou o
tipo.
