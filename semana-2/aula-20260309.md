## Programação Estruturada (slides-21)

### As três construções fundamentais

Breve introdução à programação estruturada: todo programa pode ser
construído a partir de três estruturas básicas:

- **Sequência**: instruções executadas uma após a outra, na ordem em
  que aparecem.
- **Decisão**: desvio no fluxo de execução baseado em uma condição
  (se a condição é verdadeira, executa um bloco; caso contrário,
  outro).
- **Repetição** (iteração/laço): execução repetida de um bloco de
  instruções, um número pré-determinado de vezes ou enquanto uma
  condição for satisfeita.

Fluxogramas são representações gráficas dessas estruturas: caixas para
instruções, losangos para decisões e setas indicando a ordem de
execução.

## Tipos, variáveis e literais em C (slides-22)

### Variáveis e memória

Variáveis são nomes associados a posições na memória. Cada variável
tem um nome (identificador), um valor e um endereço na memória
(acessível via `&`).

É importante distinguir três momentos na vida de uma variável:

1. **Declaração** — informa ao compilador o nome e o tipo da
   variável. O compilador reserva espaço na memória, mas o conteúdo
   é indeterminado ("lixo"):
   ```c
   int x;        // x existe, mas seu valor é lixo
   ```
2. **Inicialização** — atribuição de valor no momento da declaração:
   ```c
   int x = 10;   // declaração + inicialização
   ```
3. **Atribuição** — alteração do valor de uma variável já declarada:
   ```c
   x = 20;       // atualiza o valor de x
   ```

Atenção: em C, variáveis locais **não** são inicializadas
automaticamente com zero. Ler uma variável local não inicializada é
**comportamento indefinido** — um erro comum de iniciantes. A flag
`-Wall` do compilador ajuda a detectar esses casos.

O operador `sizeof` pode ser usado para verificar quantos bytes cada
tipo ocupa na máquina: `sizeof(int)` retorna `4` na maioria das
plataformas.

### Tipos de dados

Em C, cada tipo ocupa uma quantidade fixa de bytes:

- `bool` — 1 byte
- `char` — 1 byte
- `short` — 2 bytes
- `int` — 4 bytes (32 bits, até $2^{32}$ valores diferentes)
- `long` — 8 bytes
- `float` — 4 bytes
- `double` — 8 bytes

O tipo determina como o computador interpreta os bytes armazenados na
memória.

### Literais

Literais são notações para representar valores fixos diretamente no
código fonte. Cada literal tem um tipo implícito:

- **Literais inteiros**: `42` (decimal), `052` (octal, prefixo `0`),
  `0x2A` (hexadecimal, prefixo `0x`). Sem sufixo, o tipo é `int`.
  Sufixos: `U` (unsigned), `L` (long), `LL` (long long).
- **Literais de ponto flutuante**: `3.14`, `314e-2`, `1.3E2`. Sem
  sufixo, o tipo é `double` (não `float` — uma surpresa frequente
  para iniciantes). Sufixo `f` para `float`, `L` para `long double`.
- **Literais de caractere**: `'a'`, `'\n'`, `'\0'`. Em C, estes têm
  tipo `int` (diferente de C++, onde têm tipo `char`).
- **Literais de string**: `"hello"` — armazenados como um array de
  `char` terminado por `'\0'`.

### Identificadores (nomes de variáveis)

Regras para nomes de variáveis:

- Formados por letras, dígitos e `_`, começando por letra ou `_`.
- Maiúsculas e minúsculas são distintas (`x` e `X` são diferentes).
- Não podem ser palavras reservadas (`int`, `if`, `else`, `true` etc).
- Escolher nomes significativos para clareza do código.

Exemplos de erros: `2var` (começa com dígito), `int` como nome de
variável (palavra reservada).

### Inicialização e atualização de variáveis

Inicializar é declarar uma variável e atribuir um valor ao mesmo
tempo. O valor poderá ser um literal ou uma expressão com literais e
outros identificadores.

```c
int x = 10;
```

Atualizar é modificar o valor de uma variável já declarada:

```c
int x = 10;
x = 20;
```

Na primeira linha, `x` é criado na memória com valor `10`. Na segunda
linha, o conteúdo da memória associada a `x` é sobrescrito com `20`.
O valor anterior (`10`) é perdido.

### O operador de atribuição

O `=` em C é o operador de atribuição, **não** é teste de igualdade.

```c
s = 10 + n;
```

A execução segue dois passos: primeiro avalia a expressão à direita do
`=`, depois atribui o resultado à variável à esquerda.

#### Entendendo `x = x + 10`

Este statement é fundamental para entender como a atribuição funciona:

1. **Acesso à memória (lado direito)**: o `x` à direita do `=`
   significa "leia o valor atualmente armazenado na posição de memória
   chamada `x`".
2. **Avaliação da expressão**: o operador `+` é identificado e a soma
   é calculada — o valor lido de `x` é somado com `10`.
3. **Atribuição (lado esquerdo)**: o resultado da expressão é
   armazenado na posição de memória chamada `x`, sobrescrevendo o
   valor anterior.

Exemplo concreto:

```c
int x = 5;
x = x + 10;
```

Após a primeira linha, `x` contém `5`. Na segunda linha: lê-se `5` da
memória, calcula-se `5 + 10 = 15`, e armazena-se `15` na memória.
Agora `x` contém `15`.

O lado direito é completamente avaliado **antes** de o resultado ser
escrito no lado esquerdo. Por isso o `x` pode aparecer dos dois lados
do `=` sem contradição.

#### Lvalue e rvalue

Para entender a atribuição mais precisamente, é útil pensar nos dois
"papéis" que uma variável pode desempenhar:

- **lvalue** ("locator value"): refere-se a uma posição na memória.
  Uma variável usada à **esquerda** do `=` é um lvalue — indica
  *onde* armazenar o resultado.
- **rvalue**: refere-se a um valor temporário. Uma variável usada à
  **direita** do `=` é um rvalue — indica que queremos *ler* o valor
  armazenado naquela posição.

```c
int x = 10;
int y = x;      // x é rvalue (lê o valor 10)
x = x + 1;      // x à esquerda é lvalue, x à direita é rvalue
```

Nem toda expressão pode ser lvalue. `5 = x` é um erro porque `5` não
tem posição de memória — é apenas um valor.

### Expressões

Uma **expressão** combina literais, variáveis e operadores para
produzir um valor. Toda expressão tem um **valor** e um **tipo**. Um
literal sozinho (`42`) já é uma expressão. Uma atribuição (`x = 5`)
também é uma expressão — seu valor é o valor atribuído. Por isso é
possível encadear: `a = b = c = 0` atribui `0` a todas as variáveis.

#### Conversão implícita de tipos

Quando uma expressão mistura tipos diferentes, o C realiza conversões
automáticas. A regra geral: o operando de "menor" tipo é promovido
para o "maior":

`char` → `short` → `int` → `long` → `float` → `double`

Exemplo clássico de truncamento:

```c
int x = 3.9;    // x recebe 3, a parte fracionária é descartada
```

Exemplo de divisão inteira (que veremos nos próximos slides):

```c
int a = 5 / 3;  // resultado é 1, não 1.666...
```

Para obter o resultado com casas decimais: `5.0 / 3` ou fazer um
*cast* explícito.

### Erros comuns de iniciantes

- **Variável não inicializada**: usar uma variável local antes de
  atribuir um valor. O conteúdo é lixo.
- **Confundir `=` com `==`**: `if (x = 5)` atribui 5 a x (e sempre é
  verdadeiro), enquanto `if (x == 5)` testa igualdade.
- **Divisão inteira**: `5/3` resulta em `1`, não em `1.666...`.
- **Overflow**: atribuir `300` a um `char` (que comporta até 127 com
  sinal) causa comportamento inesperado.
