## Bubble sort — `src/bubble.c`

O **bubble sort** percorre o array repetidamente, comparando pares
adjacentes e trocando-os quando estão fora de ordem. O efeito é que,
após cada passagem, o maior elemento ainda não posicionado "sobe"
(como uma bolha) até o final.

### A função `swap`

```c
void swap(int* n, int* m) {
  int t;
  t = *n;
  *n = *m;
  *m = t;
}
```

A variável temporária `t` é essencial: sem ela, escrever `*n = *m` destruiria o
valor original de `*n` antes de salvá-lo.

### O algoritmo com duas otimizações

```c
void bubble_sort(int a[], int n) {
  bool s = true;
  while (s) {
    s = false;
    for (int i = 1; i < n; i++) {
      if (a[i] < a[i - 1]) {
        swap(&a[i], &a[i - 1]);
        s = true;
      }
    }
    n--;
  }
}
```

Duas otimizações estão presentes:

**Flag `s` (early exit)**: `s` começa `false` antes de cada passagem. Se nenhuma
troca ocorrer durante a passagem inteira, o array já está ordenado e o laço
`while` termina. No melhor caso (array já ordenado) isso reduz o custo para
$O(n)$ — uma única passagem sem trocas.

**`n--`**: após cada passagem, o maior elemento não-posicionado já se encontra
em `a[n-1]`. Não há motivo para compará-lo novamente. Decrementar `n` encurta
o laço interno a cada rodada.

### Exemplo no quadro com 8 elementos

Partindo de `[5, 3, 8, 1, 9, 2, 7, 4]`:

```
passagem 1: [3, 5, 1, 8, 2, 7, 4, 9]   →  9 está na posição final
passagem 2: [3, 1, 5, 2, 7, 4, 8, 9]   →  8 está na posição final
passagem 3: [1, 3, 2, 5, 4, 7, 8, 9]   →  7 está na posição final
...
```

A cada passagem o segmento não-ordenado encolhe em 1. Com `n--`, o laço interno
executa `n-1`, `n-2`, … comparações nas passagens sucessivas.

### Complexidade

No pior caso (array em ordem inversa) todas as passagens acontecem e há troca
a cada comparação:

$$T(n) = (n-1) + (n-2) + \cdots + 1 = \frac{n(n-1)}{2} = O(n^2)$$

É a mesma classe do selection sort, mas o bubble sort tem a vantagem do early
exit: no melhor caso é $O(n)$.


## Merge sort — `src/merge.c`

O **merge sort** é um algoritmo de divisão e conquista com complexidade $O(n \log n)$
no pior caso — uma melhoria significativa sobre os algoritmos $O(n^2)$.

### Estratégia

1. **Dividir**: se o segmento tem mais de um elemento, encontre o meio `m`.
2. **Conquistar**: ordene recursivamente as metades esquerda e direita.
3. **Combinar**: mescle as duas metades ordenadas numa única sequência ordenada.

O caso base é um segmento de tamanho 1 (um único elemento já está "ordenado").

### `merge_sort` — a recursão

```c
void merge_sort(int ns[], int i, int j) {
  if ((j - i) == 0)
    return;                   // caso base: segmento de 1 elemento

  int m = (i + j) / 2;
  merge_sort(ns, i, m);       // ordena metade esquerda
  merge_sort(ns, m + 1, j);   // ordena metade direita
  merge(ns, i, m, j);         // combina
}
```

Os índices `i` e `j` delimitam o segmento atual. `m` é o ponto de divisão.

### `merge` — a operação central

```c
void merge(int ns[], int l, int m, int u) {
  int t[u - l + 1];
  for (int k = 0, a = l, b = m + 1; k < (u - l + 1); k++) {
    t[k] = (a > m   ? ns[b++]
           : b > u  ? ns[a++]
           : ns[a] < ns[b] ? ns[a++] : ns[b++]);
  }
  for (int i = l; i <= u; i++)
    ns[i] = t[i - l];
}
```

A função recebe duas metades adjacentes já ordenadas: `ns[l..m]` e `ns[m+1..u]`.
O array auxiliar `t` recebe os elementos em ordem crescente, comparando as
"cabeças" das duas listas a cada passo:

- Se a lista esquerda esgotou (`a > m`): copia do restante da direita.
- Se a lista direita esgotou (`b > u`): copia do restante da esquerda.
- Caso contrário: copia o menor entre `ns[a]` e `ns[b]`.

Ao final, `t` é copiado de volta para o segmento original `ns[l..u]`.

### Exemplo no quadro com 8 elementos

Partindo de `[5, 3, 8, 1, 9, 2, 7, 4]`, a árvore de chamadas recursivas fica:

```
merge_sort(0, 7)
  merge_sort(0, 3)
    merge_sort(0, 1)
      merge_sort(0, 0)  → [5]
      merge_sort(1, 1)  → [3]
      merge(0, 0, 1)    → [3, 5]
    merge_sort(2, 3)
      merge_sort(2, 2)  → [8]
      merge_sort(3, 3)  → [1]
      merge(2, 2, 3)    → [1, 8]
    merge(0, 1, 3)      → [1, 3, 5, 8]
  merge_sort(4, 7)
    ...                 → [2, 4, 7, 9]
  merge(0, 3, 7)        → [1, 2, 3, 4, 5, 7, 8, 9]
```

A profundidade da árvore é $\log_2 n$. Em cada nível, o trabalho total de
`merge` é $O(n)$ (cada elemento é copiado uma vez por nível). Portanto:

$$T(n) = O(n \log n)$$

### `stderr` para depuração

O código usa `fprintf(stderr, "debug: ...")` para emitir rastreamento sem
misturar com a saída de dados em `stdout`. Isso permite redirecionar a saída
útil para um arquivo enquanto as mensagens de debug aparecem no terminal:

```
./merge < dados.txt > saida.txt
```

As mensagens de debug vão para o terminal; a saída ordenada vai para `saida.txt`.


## Comparativo dos algoritmos vistos

| Algoritmo      | Melhor caso   | Pior caso     | Estável? |
|----------------|---------------|---------------|----------|
| Selection sort | $O(n^2)$      | $O(n^2)$      | não      |
| Bubble sort    | $O(n)$        | $O(n^2)$      | sim      |
| Merge sort     | $O(n \log n)$ | $O(n \log n)$ | sim      |

Um algoritmo de ordenação é **estável** quando elementos iguais mantêm a ordem
relativa original. O merge sort é estável porque, no empate (`ns[a] == ns[b]`),
a implementação prioriza o elemento da metade esquerda (`ns[a++]`), preservando
a ordem original.
